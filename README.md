# 慢病用药小管家

美团校招 AI 编程考察演示作品：一个纯前端慢病复诊周期用药管理原型。

## 技术栈

- React
- Vite
- TypeScript
- Tailwind CSS
- localStorage

## 本地运行

```bash
npm install
npm run dev
```

## 构建检查

```bash
npm run lint
npm run build
```

## 核心功能

- 首页：评委 3 分钟体验路径、四个指标卡、低库存信息、复诊倒计时。
- 今日打卡：支持待服用、已服用、稍后提醒、已漏服；已服用后扣减演示库存。
- 幂等扣减：同一任务已服用后不会重复扣减库存，库存不会变成负数。
- 用药计划：展示“降压药 A / 降糖药 B / 调脂药 C”等演示样例，弱化真实处方药名。
- 库存续方：基于 `stockCount / dailyUsage` 估算剩余天数，低库存有文字和颜色标签。
- 用药助手：三个规则流程按钮，演示口语建档、复诊摘要和医疗越界拒答。
- 未来设想：线上零售购药提醒、线上复诊预约等仅展示说明，不接真实接口。
- 安全声明：不诊断、不处方、不停药/换药/加量，不上传真实健康数据。

## 安全边界

当前作品为校招演示版，所有用药计划、打卡记录和复诊信息仅保存在浏览器 localStorage 中，不上传服务器，不接入真实医疗、药房或支付接口。产品仅用于用药计划管理和健康记录整理，不提供诊断、处方、停药、换药或剂量调整建议。

## 部署前检查

- `npm run lint`
- `npm run build`
- 手机端检查无横向滚动
- 点击“已服用”后验证库存只扣一次
- 点击“漏服”后验证出现“请勿自行补服双倍剂量”
- 点击未来设想按钮验证不跳转真实服务
- 点击“重置为演示样例”恢复默认数据

## UI 风格说明

当前版本已按“患者关怀版”调整：

- 使用暖白、米色、柔和橄榄绿和浅琥珀色，弱化冰冷医疗科技感。
- 首页改为患者问候和“今天还需关注”的视角，不再强调内部评委语言。
- 底部 Tab 更适合移动端单手操作。
- 按钮、卡片、弹窗均使用柔和圆角和暖色阴影。
- 文案更偏照护语气，同时保留医疗安全边界。


## 纯前端说明

当前最终方案采用纯前端实现，不接入真实 AI API，也不需要后端代理。用药助手使用本地安全模板演示：

- 口语描述整理成候选用药计划
- 复诊前用药记录摘要
- 复诊问题清单
- 停药/换药/加量等医疗越界问题安全拒答

这样部署最稳：可以直接作为静态网页部署，不会泄露 API Key，也不会因为外部服务不可用影响评委体验。

## 近 7 天记录

首页的“近 7 天记录”不再是写死的 86%。当前版本会把最近 7 天的计划次数和完成次数保存在浏览器本地，并计算本周完成率。

- 默认示例：前 6 天有预置记录，今天从 0/4 开始。
- 点击今日“已服用”会同步增加今天完成次数。
- 首页“近 7 天用药记录”卡片支持用 `+ / -` 补记过去 7 天完成次数。
- 数据只保存在浏览器本地，重置示例会恢复默认 7 天记录。
